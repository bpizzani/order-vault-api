Admin Console

Create users and manage subscriptions against your backend endpoints. These forms use GET requests with query parameters to match your routes.

Requests will be sent to {baseUrl}/auth/create-user, {baseUrl}/auth/create-subscription and {baseUrl}/update-subscription-limit (adjust the prefix below if needed).
Change if your blueprints are mounted elsewhere.

1. Create User

GET /create-user Query params
201 → {"message": "✅ Created user ..."} • 400 → {"error": "..."}

2. Create Subscription

GET /create-subscription Query params
201 → {"message": "✅ Created subscription ..."} • 400/403 → {"error": "..."}

Update Subscription Limits

GET /update-subscription-limit Query params
200 → {"message": "✅ Updated subscription ..."} • 404 → {"error": "No active subscription ..."} • 400/500 → {"error": "..."}

3. Create / Update Tenant

POST /admin/upsert-tenant JSON body
201/200 → {"message": "✅ Tenant created/updated", "client_id": "..."} • 4xx/5xx → {"error": "..."}

Heads-up: Using GET for actions that mutate data exposes sensitive params in URLs and logs. Consider switching to POST with JSON + CSRF in production.

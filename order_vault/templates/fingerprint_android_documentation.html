Requirements

  • Android: minSdk 24+, targetSdk 36
  • Kotlin JVM target: 11
  • HTTPS access to api.rediim.com

2) Get your keys

You’ll receive an API Key and Client ID. Store them securely (remote config, encrypted storage, or build configs).

3) Capture fingerprint early

Call the SDK as soon as possible (e.g., right after login or on your first screen):

import com.rediim.sdk.RediimFingerprint

RediimFingerprint.sendFingerprint(
    context = this,
    apiKey = "YOUR_API_KEY",      // replace
    clientId = "YOUR_CLIENT_ID",  // replace
    userId = currentUserIdOrNull, // optional but recommended
    promocode = currentPromoOrNull,
    callback = object : RediimFingerprint.Callback {
        override fun onSuccess(visitorId: String, localSessionId: String) {
            cacheVisitorId(visitorId)
            cacheLocalSessionId(localSessionId)
        }
        override fun onError(error: String) {
            // Don't block UX; log and optionally retry later
        }
    }
)
Returns: visitorId (stable per‑device) and localSessionId (per‑session).

4) Monitoring your traffic in the Rediim Fingerprint Dashboard

Once you’ve integrated the fingerprint snippet on your app, you can monitor device and session activity in the Rediim Fingerprint Dashboard.

  • See real-time visitorId and localSessionId creation events.
  • Search by user_id to find all devices used by the same account.
  • Search by device_id (visitorId) to see how many accounts use the same device.
  • Filter by promocode to detect promo abuse patterns.

To get meaningful insights:

  • Always pass a user_id to sendFingerprint() when known.
  • If applicable, pass the promocode offered or used at the time of fingerprinting.
  • Ensure your frontend sends the same device_id and local_session_id to your backend for risk evaluation calls.

With these identifiers in place, the dashboard will let you monitor traffic, detect duplicate devices across users, and catch suspicious promo usage.

6) Minimal Activity example

class MainActivity : AppCompatActivity() {
    private var visitorId: String? = null
    private var localSessionId: String? = null
    private var userId: Int? = null
    private var promoCode: String? = null

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        setContentView(R.layout.activity_main)

        userId = (0..1000).random()
        promoCode = null // or your own promo logic

        RediimFingerprint.sendFingerprint(
            context = this,
            apiKey = "YOUR_API_KEY",
            clientId = "YOUR_CLIENT_ID",
            userId = userId,
            promocode = promoCode,
            callback = object : RediimFingerprint.Callback {
                override fun onSuccess(vId: String, sessId: String) {
                    visitorId = vId
                    localSessionId = sessId
                }
                override fun onError(error: String) { /* log + retry later if needed */ }
            }
        )
